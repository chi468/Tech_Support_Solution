서버(Server) : 서비스를 제공하는 컴픁

클라이언트(Client) : 서비스를 제공하는 컴퓨터

-> 서비스를 제공하는 프로그램

-> 서비스를 사용하는 프로그램


#네트워크 모델

1.서버기반 모델

- 장점

-> 안정적인 서비스 제공가능
-> 데이터 관리가 용이
-> 보안성도 높음

- 단점
-> 구축 및 유지보수에 비용이 많이 듬 ex)웹하드

2. Peer to Peer(P2P)

-장점
-> 가격 저렴
-> 자원 활용 극대화

- 단점
->자원 관리 어려움
-> 보안에 취약
Ex) 토렌트



-호스트(Host)
 -> 네트워크 가능한 하나의 통신기기
 -> 31대 컴퓨터 -> 31대의 호스트

-IP 주소(IP Address)

 -> 네트워크상에서 호스트를 구별하는데 사용한 고유한 주소


Java에서는
InetAddress 클래스를 이용하여 IP주소 관리



http:// 192.168.30.6:8080

8080-> 포트번호(서비스번호)
-> 관련 클래스 -> java.net.URL

프로토콜

-> 호스트간의 정보를 주고 받을 때 통신 방법에 대한 규약
-> 통신 규약
-> 전달 방식, 통신방식, 데이터 형식,  검증법 등

HTTP(HyperText TranseferProtocol) -> 80
-> 인터넷에서 웹서버와 사용자의 브라우저 사이에 문서를 전송하기 위해 사용하는 통신규약

FTP(File Transfer Protocol)
-> 호스트간의 파일전송을 위한 통신규약


Telnet(Telecommunication network) -> 23
-> 원격자의 호스트를 접속할 때 사용하는 프로토콜

SSH
-> Telnet와 비슷하나 암호화 처리가 되어있음
